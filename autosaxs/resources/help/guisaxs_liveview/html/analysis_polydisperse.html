Polydisperse analysis

Open the Polydisperse analysis window from the right-column icon (same chrome pattern as monodisperse). While it is open, new TIFFs run that chain automatically. Guinier for this window is independent of monodisperse and is stored under guinier_poly/.

Window panes

  1. Guinier — runs fit_guinier independently for display / quality. Interval edits do not feed fit_sizes (polydisperse Guinier is often poor; that is expected).
  2. fit_sizes (D(R)) — ATSAS GNOM system=1 (spheres). Default first=1 (editable). Shows I(q) and D(R) from source data (.out, D(R) CSV), plus Shannon s_min diagnostics. Param edits re-run fit_sizes (debounced).
  3. fit_mixture — optional (radio None / Mixture; default None). When enabled, auto TIFF jobs include fit_mixture. r_max / poly_max start as (auto) (skill derives them; controls update after the run). Use the model selector to explore individual MIXTURE fits (best selected by default).

Outputs (per TIFF)

  • guinier_poly/<stem>/ — Guinier region YAML
  • fit_sizes/<stem>/ — GNOM .out, D(R) CSV, quality YAML
  • mixture/<stem>/ — per-model work dirs, mixture_results.csv (when mixture enabled)

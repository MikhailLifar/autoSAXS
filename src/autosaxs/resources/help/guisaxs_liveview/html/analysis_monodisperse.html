Monodisperse analysis

Open the monodisperse analysis wizard from the right-column icon (or focus it if already open). While the window is open, new TIFFs run Guinier → GNOM → optional shape automatically. Selecting BODIES, DAMMIF, or DENSS is configuration for automatic TIFF processing — it does not pause the queue. Guinier artifacts are stored under guinier_mono/ (separate from polydisperse).

Guinier

fit_guinier. Interval edits pause auto-processing and re-run Guinier then GNOM for the current profile (shape is not chained from this manual edit).

p(r)

fit_distances (AUTORG + GNOM). Wizard options: fit_distances/fit_distances.conf. Per-TIFF results: fit_distances/<stem>/ — fit PNG, p(r) plot, GNOM .out. Editing GNOM parameters re-runs GNOM only; use Re-run shape if you also need DAMMIF/BODIES/DENSS again.

DAM

When DAMMIF is selected, automatic TIFF jobs run Guinier → GNOM → model_dam (FAST) on the GNOM output. If you select DAMMIF while a TIFF job is already mid-run (Guinier/GNOM), a shape follow-up job runs after that job finishes. Additional folder: dammif/<stem>/ — bead model, CIF, 3D preview.

DENSS

When DENSS is selected, automatic TIFF jobs run Guinier → GNOM → model_density. Default protocol is pilot (single reconstruction). average/refined run denss-all (and refine). The 3D view is a dark point cloud (ρ ≥ 15% of max): point density, cold color, and opacity encode electron density; fullscreen dialog can toggle color/opacity to the ensemble σ map when present. Per-TIFF: denss/<stem>/ — MRC density, optional σ map, I(q) fit.

Primitives

When BODIES is selected, automatic TIFF jobs include model_bodies after GNOM. Shape list: model_bodies/model_bodies.conf. Per-TIFF: model_bodies/<stem>/ — CSV, fit PNGs, 3D for best shape.
